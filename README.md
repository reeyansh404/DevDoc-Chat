# DevDoc Chat

A full-stack RAG (Retrieval Augmented Generation) chatbot. Upload any PDF and ask 
questions about it in plain English — answers are generated by GPT, grounded only in 
the content of your document.
 
**Live demo:** https://devdoc-chat.netlify.app/
**API docs:** https://devdoc-chat.onrender.com/

> Note: the backend runs on a free tier and sleeps after inactivity, so the first 
> request may take ~50 seconds to wake up.

## How It Works

When you upload a PDF, the app extracts the text and splits it into smaller chunks. 
Each chunk is converted into a vector embedding (a numerical representation of its 
meaning) and stored in a ChromaDB vector database.

When you ask a question, the question is also converted into a vector. The app 
searches the database for the chunks whose vectors are closest in meaning to your 
question, then sends those chunks along with your question to GPT. GPT answers using 
only that retrieved context — so the response is grounded in your document rather 
than the model's general knowledge.

## Tech Stack

- **FastAPI** — Python backend serving the upload and ask API routes
- **LangChain** — orchestrates the RAG pipeline (loading, chunking, embedding, retrieval)
- **ChromaDB** — vector database that stores and searches the document embeddings
- **OpenAI (GPT-4o-mini)** — generates the final answers
- **HTML / JavaScript** — frontend for uploading documents and asking questions

## How to Run

Backend:

```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload
```

Frontend — open `frontend/index.html` in your browser.

You'll need an OpenAI API key in a `.env` file inside the backend folder:

```
OPENAI_API_KEY=your_key_here
```
